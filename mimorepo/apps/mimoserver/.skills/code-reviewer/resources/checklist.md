# 代码审查检查清单

## 通用检查项

### 代码结构
- [ ] 代码遵循项目既定的目录结构
- [ ] 文件命名符合规范
- [ ] 模块职责单一、清晰
- [ ] 避免循环依赖

### 命名规范
- [ ] 变量名有意义且准确
- [ ] 函数名使用动词开头
- [ ] 常量使用全大写
- [ ] 避免使用缩写和不常见术语
- [ ] 遵循语言特定的命名约定

### 代码质量
- [ ] 避免重复代码 (DRY 原则)
- [ ] 函数长度适中 (< 50 行)
- [ ] 嵌套层级不超过 3 层
- [ ] 魔法数字已提取为常量
- [ ] 注释解释"为什么"而非"是什么"

### 错误处理
- [ ] 所有异步操作有错误处理
- [ ] 提供有意义的错误信息
- [ ] 避免吞没错误
- [ ] 边界条件已处理

### 测试
- [ ] 包含单元测试
- [ ] 测试覆盖关键路径
- [ ] 测试命名清晰描述场景
- [ ] Mock 外部依赖

## 审查反馈原则

1. **具体明确**: 指出具体的代码位置和问题
2. **解释原因**: 说明为什么需要修改
3. **提供建议**: 给出改进建议或参考
4. **区分优先级**: 标记必须修复 vs 可选改进
