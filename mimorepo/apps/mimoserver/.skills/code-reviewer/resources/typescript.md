# TypeScript 专项审查

## 类型安全

### 类型定义
- [ ] 避免使用 `any` 类型
- [ ] 为函数参数和返回值添加类型
- [ ] 使用 `interface` 或 `type` 定义数据结构
- [ ] 为复杂类型提取别名

### 类型断言
- [ ] 避免不必要的类型断言
- [ ] 优先使用类型守卫
- [ ] 使用 `as const` 获取精确字面量类型

### 泛型
- [ ] 合理使用泛型提高复用性
- [ ] 泛型参数有明确的约束
- [ ] 避免过度复杂的泛型

## TypeScript 特性

### 可辨识联合
- [ ] 使用可辨识联合替代可选字段
- [ ] 为联合类型提供公共属性

### 工具类型
- [ ] 使用 `Partial<T>` 处理可选更新
- [ ] 使用 `Pick<T>` 和 `Omit<T>` 提取类型
- [ ] 使用 `Record<K, V>` 定义字典类型

### 严格模式
- [ ] 启用 `strictNullChecks`
- [ ] 启用 `noImplicitAny`
- [ ] 启用 `strictFunctionTypes`
- [ ] 处理所有 `undefined` 和 `null` 情况

## 常见问题

### 类型推断
- [ ] 不要为了类型而添加不必要的类型注解
- [ ] 信任 TypeScript 的类型推断
- [ ] 在复杂场景提供显式类型

### 类型的可维护性
- [ ] 类型定义与使用位置靠近
- [ ] 导出可复用的类型
- [ ] 为类型添加 JSDoc 注释
