# React 组件审查要点

## 组件设计

### 职责单一
- [ ] 组件只做一件事
- [ ] 组件大小合理 (< 300 行)
- [ ] 避免过度嵌套

### 命名规范
- [ ] 组件使用 PascalCase
- [ ] 属性使用 camelCase
- [ ] 事件处理函数以 `handle` 开头
- [ ] 布尔属性以 `is/has/show` 开头

## Hooks 使用

### useState
- [ ] 状态尽可能简单
- [ ] 相关状态合并为一个对象
- [ ] 避免冗余状态（可由其他状态派生）

### useEffect
- [ ] 包含所有依赖项
- [ ] 清理副作用（返回清理函数）
- [ ] 避免在 effect 中执行昂贵的计算
- [ ] 区分开发/生产环境的额外渲染

### 自定义 Hooks
- [ ] 以 `use` 开头
- [ ] 可复用逻辑提取为自定义 Hook
- [ ] 参数和返回值类型明确

## 性能优化

### 避免不必要的渲染
- [ ] 使用 React.memo 包裹纯组件
- [ ] 使用 useMemo 缓存计算结果
- [ ] 使用 useCallback 稳定函数引用

### 列表渲染
- [ ] 为列表项提供稳定的 key
- [ ] 大列表使用虚拟滚动
- [ ] 避免在渲染函数中创建新对象/数组

## 常见问题

### 条件渲染
- [ ] 使用 `&&` 或三元运算符
- [ ] 避免在 JSX 中调用函数

### 事件处理
- [ ] 事件处理函数使用 useCallback 包装
- [ ] 事件对象正确类型标注

### Props 传递
- [ ] 避免深层 props 传递
- [ ] 考虑使用 Context 共享状态
- [ ] 使用 TypeScript 定义 Props 接口
