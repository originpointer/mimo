# 安全审查要点

## Web 应用安全

### 输入验证
- [ ] 所有用户输入都经过验证
- [ ] 使用白名单而非黑名单
- [ ] 验证数据类型、长度、格式
- [ ] 防止批量请求攻击

### SQL 注入
- [ ] 使用参数化查询
- [ ] 避免拼接 SQL 语句
- [ ] 使用 ORM 安全方法
- [ ] 最小化数据库权限

### XSS 防护
- [ ] 输出时进行 HTML 转义
- [ ] 使用 CSP (Content Security Policy)
- [ ] 设置正确的 Cookie 标志 (HttpOnly, Secure, SameSite)
- [ ] 避免使用 `innerHTML` 或 `dangerouslySetInnerHTML`

### 认证授权
- [ ] 密码使用 bcrypt/argon2 加密
- [ ] 实现速率限制
- [ ] 使用 HTTPS
- [ ] JWT/Session 有合理的过期时间
- [ ] 实现登出功能

### 敏感信息
- [ ] 不在日志中记录敏感信息
- [ ] 不在 URL 中传递敏感数据
- [ ] API 密钥存储在环境变量
- [ ] 不在代码中硬编码凭证

### 文件上传
- [ ] 验证文件类型（MIME + 扩展名）
- [ ] 限制文件大小
- [ ] 重命名上传文件
- [ ] 存储在 web 根目录外
- [ ] 扫描恶意文件
